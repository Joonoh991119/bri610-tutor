# v0.5 Demo — Run Book

The demo is the smallest viable slice of the [revised v0.5 plan](benchmarks/v0.5/00b_revised_plan_with_R1-R5.md). It demonstrates the load-bearing pieces — Multi-Lens Review, SymPy verifier, FSRS-6 SRS, persona narrator, harness LLM client — over a 12-card hand-curated bank covering Nernst / HH / cable theory × 4 card types.

> Goal: J confirms the foundation is sound, then auto-evolution mode iterates UI/persona/coverage until production-ready.

## What's in the demo

| Layer | Component | File / Endpoint |
|---|---|---|
| Schema | v0.5 deltas (users / mastery / figures / question_bank / srs_cards / question_review_log / lens_disagreement_log / foundation_content / analytics_events) | `pipeline/migrations/002_v05_schema.sql` |
| Verifier | SymPy preprocessor + cascade entry (10/10 tests pass) | `backend/verify/`, `tests/test_verify.py`, `POST /api/verify` |
| Harness | Per-route LLM dispatcher (paid-only-where-needed; OpenRouter→Ollama fallback; telemetry) | `backend/harness/llm_client.py` |
| Hooks | `pre_question_display`, `post_answer`, `pre_derivation`, `post_walkthrough_step` | `backend/harness/hooks.py` |
| Multi-Lens | Factual (veto) / Pedagogical / Korean / Difficulty (advisory) — convergence in ≤3 rounds | `backend/review/multi_lens.py`, `POST /api/review/multi-lens` |
| Persona | `뉴런쌤` post-processor (cheap free-tier, evolves with user state) | `backend/agents/persona_narrator.py`, `POST /api/persona/wrap` |
| SRS | FSRS-6 scheduler + queue + review | `backend/srs/scheduler.py`, `GET /api/srs/queue`, `POST /api/srs/review` |
| Seed | 12 hand-curated bank items: 3 topics × 4 card types | `scripts/seed_bank_demo.py` |
| UI | New `복습 (SRS)` tab — KaTeX render + 4 rating buttons + persona session opener | `frontend/src/components/SRSPanel.jsx` |

## Run book

```bash
# 1) Apply schema migrations (idempotent)
psql -d bri610 -U tutor -f pipeline/migrations/001_lecture_summaries.sql
psql -d bri610 -U tutor -f pipeline/migrations/002_v05_schema.sql

# 2) Install new Python deps
cd backend && pip install -r requirements.txt && cd ..

# 3) Seed the demo bank (12 items, no LLM cost)
python scripts/seed_bank_demo.py
# Optional, runs Multi-Lens Review on each item (uses OPENROUTER_API_KEY):
# python scripts/seed_bank_demo.py --review

# 4) Run backend
cd backend
export OPENROUTER_API_KEY=sk-or-v1-...
uvicorn main:app --port 8000 --reload &
cd ..

# 5) Smoke test
bash scripts/demo_smoke.sh

# 6) Frontend (existing)
cd frontend && npm run dev
# → http://localhost:3000  → click the "복습" tab
```
